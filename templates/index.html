About the Model & How It Works

This tool uses a rule-based classifier built with the Logical Analysis of Data (LAD) approach on the Cleveland Heart Disease dataset.

Training & Prediction Process – Step by Step

  1. Data Preparation
    The Cleveland dataset contains 14 features (age, sex, cp, trestbps, chol, etc.) and a binary target (0 = no disease, 1 = disease).
  2. Discretization (Binarization)
    Numeric features are converted into many binary (0/1) conditions using shallow decision trees.
    Example: "chol ≤ 219.58" becomes a binary feature (1 = yes, 0 = no).
  3. Feature Selection
    Only the most promising binary features are kept using A* search guided by cross-validated logistic regression accuracy.
  4. Rule Mining
    Maximal high-purity patterns are discovered using greedy backward elimination.
    Only rules with purity ≥ 89–96% are kept. Each rule gets a weight based on its frequency within the predicted class.
  5. Prediction
    For a new patient:
    • All features are binarized using the same thresholds.
    • We look for the rule with the highest score = overlap × rule weight.
    • If exact match → strongest confidence.
    • If no exact match → best partial match is used (always gives an answer).
  6. Output
    You get the predicted class + confidence + the exact rule that fired (or best matching one).
